
# 재수강

* C 이하의 과목이 없는 경우 -> 패스
* C 이하의 과목은 있지만 이번 학기에 개설되지 않은 경우(courseList가 없다면, 다음학기든지 안생기는 과목이든지) -> 이번 학기에 수강할 과목이 없다 메세지 보여주기 (에러코드처럼)


# 전공 필수 & 기초(동일 적용)

* "졸업사정표 상 전공필수 총족 여부 satisfied == TRUE"일 경우
  -> 이미 전공 필수 학점을 전부 채웠다는 메세지(과목 있어도 보여줄 필요 없음)
* "졸업사정표 상 전공필수 충족 여부 == FALSE && 들을 과목이 없는 경우(courseList가 빈 경우)
  -> 이번 학기에 수강할 전공필수 과목이 없다고 안내할 수 있는 메세지
* 졸업사정표에 전공필수가 없는 경우 -> 패스

=>

양수,양수,true면 이미 전공 필수 학점을 전부 채웠다 메세지 보여주기

0,0,true면 courseList가 비었으면 패스

양수,양수,false + courseList 있으면 과목 목록 보여주기

양수,양수,false + courseList 비었으면 이번 학기에 수강할 전공필수 과목이 없다 메세지 보여주기

# 전공 선택

* 다 들었을 경우 -> 전공 선택 학점을 이미 다 채웠다는 메시지(과목 있어도 보여줄 필요 없음), 화면은 패스 x

=>

true면, 이미 다 채웠다 메세지 보여주기

false면 과목 목록 보여주기

# 복수전공, 부전공

* 복수전공 or 부전공 자체가 없는 경우 -> 아예 화면도 보여주지 않고 패스
* 복수전공 or 부전공이 있는 경우
  * 필수를 이수 완료한 경우
    * 이수 완료 정보를 보여주돼 과목은 비워서(화면은 표시 -> 다 들었습니다~~ 이런식으로)
  * 선택을 이수 완료한 경우
    * 이수 완료 정보를 보여주돼 과목은 비워서(화면은 표시-> 다 들었습니다~~ 이런식으로)

=>

0,0,true => 복수전공/부전공이 없으므로 패스

복수전공의 경우:

* DOUBLE_MAJOR_REQUIRED, DOUBLE_MAJOR_ELECTIVE를 2개 comma separated로 분리해서 한번에 조회
* 필수, 선택의 학점 정보는 무조건 보여준다.
* 과목 목록이 비었다면 모두 이수했다는 메세지 표시
* course의 field 항목에 "필수" 또는 "선택"이 들어있어서 프론트에서 필터링.
* course의 targetGrades 배열의 첫 번째 요소만 사용.
  * [1,2] 면 무시하고 첫번째 학년 값에만 보여주기.

부전공의 경우:

* MINOR 카테고리를 조회
* course의 field 항목에 "필수" 또는 "선택"이 들어있어서 프론트엔드에서 필터링

# 교직이수

* 교직이수 자체가 없는 경우 -> 화면도 보여주지 않고 패스
* 교직이수가 있는 경우
  * 어짜피 이수 정보 못불러와서 화면 & 과목 무조건 보여주기

=>

0,0,true면 교직이수 없음 -> 패스

* TEACHING으로 조회
* courseList에서 field가 전공영역, 교직영역, 특성화로 필터링

# 교양필수

* "졸업사정표 상 교양필수 총족 여부 == TRUE"일 경우 -> 이미 교양필수 학점을 전부 채웠다는 메세지
* "졸업사정표 상 교양필수 총족 여부 == FALSE" && 보내줄 과목 정보가 하나도 없는 경우(수강 대상 맞는 과목이 없는 경우)
  * 수강 대상이 맞는 과목이 없음을 알리는 메세지

=> true면 무조건 이미 교양필수 학점을 전부 채웠다

false인데 courseList가 비었다면 수강 대상이 맞는 과목이 없음
