### 복수전공&부전공(다른 점 없음)

복수전공으로 표기 통일하겠습니다.

목적
사용자가 이번 학기에 들어야하는 복수전공 과목을 알 수 있게 하기 위함

Input
토큰(사용자의 학번, 학과, 학년, 이때가지 수강한 과목, 복수전공 유무, 복수전공 학과 이름)

Logic

1. 해당 복수전공의 복필 과목을 조회한다.
2. 해당 복수전공의 복선 과목을 조회한다.
3. 사용자가 이미 들은 과목을 제외한다.

Output

1. 사용자의 복필 & 복선 이수 현황
2. 복필 과목
   1. 학년 구분 없이 분반으로 묶어서 전달
   2. 과목 정보(과목명, 교수, 시간)
3. 복선 과목
   1. 학년 구분 있이 학년으로 묶고 분반으로 묶어서 전달
   2. 과목 정보(과목명, 교수, 시간)

Edge Case

* 복필을 이수 완료한 경우
  * 이수 완료 정보를 그대로 보내주돼 과목은 비워서 보내주세요(화면은 표시할 계획입니다 -> 다 들었습니다~~ 이런식으로)
* 복선을 이수 완료한 경우
  * 이수 완료 정보를 그대로 보내주돼 과목은 비워서 보내주세요(화면은 표시할 계획입니다 -> 다 들었습니다~~ 이런식으로)
* 졸업사정표에 복필 또는 부필이 없는 경우도 고려해야할 것 같네요
  * 복필 또는 부필이 있는지 없는지 여부를 항상 보내줘야할 것 같습니다!
