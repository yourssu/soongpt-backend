## 전공기초

목적
사용자가 이번 학기에 들어야하는 전공 기초 과목을 알고, 지나간 전공 기초 과목을 알기 위함

Input
토큰(사용자의 학번, 학과, 학년, 이때가지 수강한 과목(전체, 원래 전공 선택이였던 과목이 현재는 전공기초로 변경된 경우를 대비하기위해), 졸업사정표의 전공기초 학점&충족 여부)

Logic

1. 이번 학기에 열린 사용자의 학과 전공기초 과목 조회
2. 이미 수강한 과목 제외(성적 무관)
3. 남은 과목 3가지 케이스로 분리
   1. 과목 학년 < 사용자의 학년
   2. 과목 학년 = 사용자의 학년
   3. 과목 학년 > 사용자의 학년

Output

1. 사용자의 전공 기초 과목 이수 현황(ex. 15/20)
   1. 졸업사정표 "전공기초" section에서 가져오기
2. Logic-3-a에 해당하는 과목(이 경우에는 수강 대상이 맞지 않을텐데 잘 패스해서 넘겨주실 수 있을까요?)
   1. 수강 과목 정보(과목 이름, 교수, 시간)-분반 전체 묶어서
   2. 원래 수강해야하는 학년
   3. 미수강 태그를 붙여서(어떤 식으로 해야할지는 잘 모르겠네요..)
   4. 대상외수강제한일 경우에는 태그 붙여서
3. logic-3-b에 해당하는 과목
   1. 수강 과목 정보(과목 이름, 교수, 시간)-분반 전체 묶어서
   2. 정상 태그를 붙여서

Edge Case

* "졸업사정표 상 전공기초 총족 여부 == TRUE"일 경우 -> 이미 전공 기초 학점을 전부 채웠다는 메세지
  * TRUE 여부를 가장 먼저 검사하고 패스
* "졸업사정표 상 전공기초 충족 여부 == FALSE && logic-3-a와 logic-3-b에 해당하는 과목이 없는" 경우 -> 이번 학기에 수강할 전공기초 과목이 없다고 안내할 수 있는 메세지
* 졸업 사정표에 전공기초가 존재하지 않는 경우 -> 없다고 알려주세용
