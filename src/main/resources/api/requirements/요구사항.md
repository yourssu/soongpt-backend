근데 이거 좋은데 교필도 비슷하고 교선도 비슷하고 재수강도 비슷해 심지어 엣지케이스도 공통이야.이거 고려해서전체적으로 그냥 엔드포인트는 하나로 두되, 이수구분(재수강, 전필, 전기, 전선, 교필, 복수전공, 부전공, 교직이수) 이렇게 나눠서보면되거든??

전체적으로 공통되는 부분이있는데 너가 이미 짠 추상화로직을 어떻게 디벨롭시키면 좋을지 알려줘.

1. 해당 이수구분의 개설 과목 조회 (이번 학기 개설 + 사용자가 들어야하는) -> 이미 있는 로직활용
2. 사용자 이수 코드로 수강한 과목 제외 (여기서 rusiant-serivce에서 가져온 정보가 활용됨)
3. 분반별 그룹핑
4. 응답 생성

사용자가 들어야하는 = 사용자가 n학년이라 가정했을때 1~n-1학년 이수구분의 과목들(LATE) + n학년의 이수구분과목들(ON_TIME)

근데 응답도 전체적으로 비슷해.

지금 무조건 공통적으로 띄워야하는 정보

* 지금까지 졸업사정표의 정보 (현재 이수구분의 n학점 중 m학점을 이수하였다 3가지필드로 나눠서 보내기

required, completed, satisfied) - 재수강빼고 나머지이수구분 다 해당

* 과목(카드썸네일에 띄움)

  * 과목이름(선형대수)
  * 해당과목을 가르치는 교수들(과목의 분반들의 교수들을 다 합쳐서 중복제거한)
  * 학점(3학점, course의 point)
  * 개설학과 (course의 `department`) - 전공일때만 띄우기 근데 일단 다 보내주긴할듯?
  * 타겟(텍스트)
  * course timing (LATE, ON_TIME)
* 분반정보(썸네일에서 클릭했을때)

  * 분반이름 (01,02,03)
  * 분반별 교수님
  * 분반별 courseTime(월 10:00 ~ 11:15, 수 10:00 ~ 11:50)
  * 분반별 대상외수강제한 여부(target의 `isStrict`)

엣지케이스도 공통적인게 있어.

* 이미 이수구분을 다 들어서 들을 필요가 없는 경우.  satisfied=true
* 이수구분을 다 안듣긴했는데 들어야하는 과목이 이번학기에 개설되는 과목이 없는 경우. satisfied=false

근데 이제 여기서 좀 이수구분별로 특이점이 있어.

전공선택 - 학년별, 타전공 영역으로 구분해서 보내야해. 현재 학년도 같이 보내야해.

교양필수 - 이건 조금 다른게 학년별 분야가 있어.

1. 해당 이수구분의 개설 과목 조회 (이번 학기 개설 + 사용자가 들어야하는) -> 이미 있는 로직활용
2. 사용자 이수 코드로 수강한 과목 제외 (여기서 rusiant-serivce에서 가져온 정보가 활용됨)
3. 분반별 그룹핑
4. 응답 생성

이 상황에서 2번을 진행할때, 분반만 제외시키는게 아니라 분야자체를 제외시키는거로 진행해야할거같아.

ex) SW와AI(1학년)   글로벌시민의식(2학년) 글로벌소통과언어(2학년) 창의적사고와혁신(3학년) 이런식으로.

그리고 미수강분야(LATE)에 대한 과목정보는 아예 보내지않고, 대신 미수강 분야를 따로 텍스트로 전달해야해.

ON_TIME에 대한 과목정보는 위의 공통사항과 똑같이 보내. (근뎨 얘도 1~4 똑같이 거치는건맞아). 그리고 얘는 분야로 구분해서 전달해야해.

교양선택 - 이것도 조금 다른게 얘도 분야가 있어.

교양필수랑 마찬가지로 2번을 진행할때, 분반만 제외시켜야대. 진행해야할거같아. 그리고 얘도 분야로 구분해서 전달해야해.
