# PT-141 비교 리포트: 소프트웨어학부 / AI융합학부 / AI소프트웨어학부 수강대상 관계

## 목적
- 세 학과(`AI소프트웨어학부`, `소프트웨어학부`, `AI융합학부`)의 `target` 데이터를 비교해
  - 차이점(누락/추가)을 확인하고
  - 수강대상 관계가 필요충분인지 판단한다.

## 비교 기준
- 비교 단위 1: **target row 기준**
  - 키: `course_code, scope_type, student_type, college_id, grade1~5, is_denied, is_strict`
- 비교 단위 2: **과목 코드(course_code) 기준**

## 집계 결과
### 1) target row 수
- AI소프트웨어학부(9): **127**
- 소프트웨어학부(5): **146**
- AI융합학부(2): **147**

### 2) 과목 코드 수(중복 분반 제외하지 않음: code 단위)
- AI소프트웨어학부(9): **115**
- 소프트웨어학부(5): **132**
- AI융합학부(2): **129**

## 포함관계(필요/충분) 판정
### row 기준
- `AI소프트웨어학부 ⊆ 소프트웨어학부` : **참** (`A_not_S = 0`)
- `AI소프트웨어학부 ⊆ AI융합학부` : **참** (`A_not_I = 0`)
- 역포함(`소프트웨어학부 ⊆ AI소프트웨어학부`, `AI융합학부 ⊆ AI소프트웨어학부`) : **거짓**
  - `S_not_A = 19`
  - `I_not_A = 20`

### course_code 기준
- `AI소프트웨어학부 ⊆ 소프트웨어학부` : **참** (`course_A_not_S = 0`)
- `AI소프트웨어학부 ⊆ AI융합학부` : **참** (`course_A_not_I = 0`)
- 역포함은 거짓
  - `course_S_not_A = 17`
  - `course_I_not_A = 14`

## 해석 (필요충분관계)
명제 `P(x): x가 AI소프트웨어학부 수강대상`, `Q(x): x가 소프트웨어학부 수강대상`, `R(x): x가 AI융합학부 수강대상`으로 보면:
- `P -> Q` 성립, `P -> R` 성립
- `Q -> P` 불성립, `R -> P` 불성립

즉,
- **AI소프트웨어학부 수강대상은 소프트웨어/AI융합 수강대상의 충분조건**
- 동시에 **소프트웨어/AI융합 수강대상은 AI소프트웨어 수강대상의 필요조건**
- 하지만 **필요충분(iff) 관계는 아님**

## 주요 차이 예시

### AI융합학부에만 있고 AI소프트웨어학부에는 없는 과목(code)

- 2150057301 AI시대의정보보안
- 2150064501 오픈소스기반고급설계
- 그 외 포함 총 14 code

### 소프트웨어학부 vs AI융합학부 상호 차이
- 소프트웨어학부만 보유: 5 code
- AI융합학부만 보유: 2 code

## 결론
- PT-141로 보정한 결과, **AI소프트웨어학부 대상은 두 학과(소프트웨어/AI융합)에서 모두 수강 가능**하도록 충족됨.
- 다만 두 기존 학과는 각자 고유 target이 있어, 세 집합은 **동치 집합이 아니라 상위집합 관계**임.
